# Git Workflow
- If working on a complex task - create atomic commits
- Stage only changes related to current task
- Write clear, descriptive commit messages

# Code Standards
- Code Language: ALL code, variables, functions, comments MUST be in English ONLY
- Use descriptive names, avoid abbreviations
- Include proper exception handling and logging
- Comment "why", not "what"
- Functions = verbs, Classes = nouns
- Follow SOLID, DRY, KISS principles
- Optimize for readability and performance
